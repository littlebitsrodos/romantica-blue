# Reference

This folder holds tracked reference material that belongs in the repository.

Use `reference/` for stable project knowledge that should travel with the codebase.
Use `archive/` for historical artifacts, diagnostics, captures, exports, and local-only storage.

## Structure

- `architecture/`: Architecture notes and follow-up sessions.
- `design/`: Design reference material.
- `testing/`: Testing reference docs.
- `assets/`: Reference assets worth keeping in the repo.

## Guidelines

- Keep files here when they help future work and should remain versioned.
- Do not use `reference/` for temporary diagnostics or machine-local clutter.
- If the artifact is historical rather than reference material, prefer `archive/`.
